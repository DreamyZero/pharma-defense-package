# Demo defense step

Этот этап связывает frontend admin-экран с backend API импорта и аудита.

## Что сделано

- Добавлен отдельный `frontend-react` на Vite + React.
- Создан `api.ts` с axios и Bearer interceptor.
- Реализованы клиентские сущности `imports` и `audit`.
- Добавлена страница `AdminPage` с KPI, таблицей импортов, кнопкой запуска ETL и журналом аудита.
- Поддержан fallback-режим, чтобы UI можно было показать даже без запущенного backend.

## Почему это важно для защиты

В дипломе отдельно описаны клиентская часть на React, маршрутизация, MobX/сущности, API-интерцепторы и административные сценарии. Этот шаг даёт уже демонстрируемый front-to-back сценарий для показа комиссии.

## Как показывать

1. Открыть frontend.
2. Показать KPI и таблицу ETL.
3. Запустить demo-job через кнопку.
4. Показать журнал аудита.
5. Пояснить, что при наличии backend запросы идут в `/imports` и `/audit`, а без backend работает fallback для офлайн-демо.
