# ETL pipeline

Этот этап переводит проект к доменной модели данных и контуру загрузки знаний.

## Что входит

- `etl/data/raw/drugs_raw.csv` — сырой демонстрационный набор.
- `etl/src/run_etl.py` — нормализация и разложение данных по сущностям.
- `etl/src/generate_neo4j_seed.py` — генерация `neo4j_seed.cypher`.
- `etl/data/processed/*.csv` — нормализованные таблицы.
- `etl/output/etl_report.csv` — отчёт по ETL.
- `etl/output/neo4j_seed.cypher` — сид для графа знаний.

## Смысл архитектуры

По диплому система должна собирать данные из ГРЛС и инструкций, унифицировать терминологию, строить семантические связи и хранить данные гибридно: PostgreSQL для пользователей и служебной части, Neo4j для графа знаний о препаратах. Этот шаг именно это и подготавливает.

## Следующий реальный этап

1. Заменить `drugs_raw.csv` на парсинг ГРЛС/XML и HTML/PDF инструкций.
2. Подключить запись processed-данных в PostgreSQL/Neo4j.
3. Запускать ETL как фоновый job и логировать статусы импорта.
